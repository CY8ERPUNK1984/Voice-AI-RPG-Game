# План решения проблем

1. **Унифицировать переменные окружения**
   - Переименовать `VITE_WS_URL` в `VITE_SOCKET_URL` в файлах примера и документации.
   - Проверить использование переменных в коде и обновить README.

2. **Использовать один экземпляр `GameSessionManager`**
   - Создать `GameSessionManager` в `GameController` и передавать его в `WebSocketServer` через конструктор, чтобы HTTP‑маршруты и WebSocket работали с общими данными.

3. **Упростить установку зависимостей**
   - Добавить в README инструкцию запускать `npm run install:all` перед разработкой или вынести установку в скрипт `postinstall` в корневом `package.json`.

4. **Проверить работу внешних API**
   - Убедиться, что в `.env` указаны корректные ключи OpenAI.
   - Добавить обработку ошибок при отсутствии ключей и информирование пользователя.

5. **Запуск тестов**
   - После установки зависимостей выполнять `npm test`.
   - Настроить CI (GitHub Actions) для автоматического выполнения тестов.

6. **Документация и примеры**
   - Внести правки в `ENV_SETUP.md` и `README.md`, чтобы сведения о переменных соответствовали коду.
